What is Kubernetes?

Kubernetes, designed by Google and currently maintained by the Cloud Native Computing Foundation, is an open-source container-orchestration system. It helps in automating application deployment, scaling, and management.

    Describe your containers and the way several you wish with a “Deployment.” Kubernetes can keep those containers running and handle deploying changes (such as change the image or application variables) with a “rollout.” You will pause, resume, and rollback changes as you prefer.

What our distribution will offer?

To turn Kubernetes into a ready-to-use box it’s should to be expanded with the following services

  • CI/CD

    GitLab is a platform for version control and collaboration in software development. GitLab CI is a built-in continuous integration tool that automates building, testing, and deploying code changes.

  • Application Management and Operations

    ArgoCD is a specialized open-source tool for deploying and managing applications in Kubernetes clusters using GitOps. The ArgoCD UI offers a user-friendly interface with visualizations and easy navigation where developers can control and change their applications behaviour and state.

  • Monitoring, Observability, Alerting

    Prometheus is a monitoring toolkit for collecting and storing time-series data from cluster component and application’s metrics, Grafana is a data visualization and monitoring platform, and Loki is a log aggregation system for searching and analyzing application and cluster logs.

  • SSO (Single Sign-on), Centralized Authentication service

    Keycloak is an open-source identity and access management (IAM) solution that provides authentication, authorization, and single sign-on capabilities (SSO) for web and mobile applications. It simplifies the management of user identities, allowing organizations to secure their applications and control access to resources efficiently.

  • Built-in Container Registry

    In Kubernetes, a Docker registry is a repository for storing container images used by Kubernetes clusters for deployment and management of containers. It facilitates the sharing and distribution of container images within the Kubernetes ecosystem.

  • GPU acceleration

    GPU acceleration in Kubernetes leverages Graphics Processing Units (GPUs) to enhance the performance of computationally intensive workloads, enabling faster execution and improved efficiency for tasks like machine learning and data processing.
